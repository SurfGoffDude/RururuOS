## Bug Tracking

### GitHub Issues

**Labels:**
- `bug` - Something isn't working
- `enhancement` - New feature request
- `documentation` - Docs improvement
- `good first issue` - Good for newcomers
- `help wanted` - Extra attention needed
- `priority:critical` - Must fix ASAP
- `priority:high` - Important
- `priority:medium` - Normal priority
- `priority:low` - Nice to have
- `package:*` - Specific package
- `workflow:*` - Specific workflow

**Issue Templates:**
- Bug Report
- Feature Request
- Hardware Compatibility Report
- Documentation Request

### Bug Report Template

```markdown
## Description
Brief description of the bug

## Steps to Reproduce
1. Step one
2. Step two
3. ...

## Expected Behavior
What should happen

## Actual Behavior
What actually happens

- RururuOS Version:
- Hardware: (output of `rururu-hwdetect`)
- Desktop: Sway/GNOME/KDE

## Logs
```
Relevant logs here
```

## Screenshots
If applicable
```

## Contributing

### Code Contributions

1. Fork repository
2. Create feature branch
3. Make changes
4. Run tests
5. Submit pull request

### Documentation Contributions

- Edit docs in `/docs` directory
- Submit PR with changes
- Translations welcome

### Testing Contributions

- Test nightly builds
- Report issues
- Verify fixes

### Community Contributions

- Help in support channels
- Write tutorials
- Create videos
- Translate content

## Governance

### Maintainers

Core team responsible for:
- Code review
- Release management
- Community moderation

### Decision Making

- Major changes discussed in GitHub Discussions
- RFCs for significant features
- Voting for controversial decisions

### Code of Conduct

Based on Contributor Covenant:
- Be respectful
- Be inclusive
- No harassment
- Constructive feedback

## Recognition

### Contributors

All contributors acknowledged in:
- CONTRIBUTORS.md
- Release notes
- Website credits

### Community Awards

- Monthly contributor highlight
- Best tutorial/guide
- Most helpful community member
